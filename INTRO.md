# e2e-solution-guard 简介

**e2e-solution-guard** 是 Kimi / Claude / Codex 的 Agent Skill，用于管理从需求到代码的全链路产物——PRD、交互设计、UI 原型、技术方案、测试报告。

## 解决什么问题

用 AI 辅助写方案时，常见的困扰：
- AI 在文档里添加了未定义的功能点或字段
- 修改 PRD 后，技术方案和测试用例没有同步更新，产物之间逐渐不一致
- 跨会话后 AI 忘记之前的约束，规则被悄悄绕过

## 怎么工作

**产物依赖网络**：每份产物文件头部声明上游来源（如 PRD 依赖顶层定义，交互设计依赖 PRD），形成可追溯的依赖链。

**变更双向约束**：
- 修改上游时，AI 会扫描下游影响，询问是否同步更新
- 修改下游时，AI 会逐条比对是否在上游定义范围内，超出则提示先回改上游

**自动审计**：内置 `doc-audit.py` 脚本，可检测编号重复、引用失效、术语不一致等问题。

## 适用场景

- 需要 AI 协助编写产品文档、技术方案
- 团队希望保持 PRD、交互、UI、技术方案之间的一致性
- 项目涉及多个角色（如消费者端 + 管理后台）或多端（Web + App）
