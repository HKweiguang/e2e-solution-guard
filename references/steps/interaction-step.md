# 交互设计模板

> 定位：定义用户怎么用产品，包括页面结构、状态流转、操作流程、异常处理、交互手势。

---

## §1 文件头部格式

```markdown
# 交互设计-{页面组名/流程名}

**覆盖功能点**：{功能编号}, {功能编号}, ...
**覆盖页面**：{页面编号}, {页面编号}, ...
**版本号**：v{主版本号}.{补丁号}
**制定日期**：YYYY-MM-DD
**作者**：姓名

**上游文档**：
> 以下仅为该步骤常见依赖的示例，实际生成时需根据项目上下文动态判断，不要机械套用。

| 文档 | 类型 | 引用范围 |
|------|------|---------|
| PRD-v{主版本号}-{模块名} | 需求输入 | {功能编号范围}, {页面编号范围} |
| 项目交互-顶层定义 | 规范继承 | {端名}导航结构、手势规则、组件行为 |
```

---

## §2 章节结构

每个页面为一个一级章节，标题格式：`## §N {页面编号} {页面名}`（如 `## §1 PAGE-001 商品列表`）。

每个页面必须包含以下子节：

| 子节 | 说明 |
|------|------|
| 信息架构 | 页面元素及层级关系 |
| 状态机 | 默认/空/加载/错误/成功状态 |
| 页面流程 | 用户从进入到离开的操作路径 |
| 异常处理 | 异常场景、系统响应、用户感知 |
| 与 PRD 对应 | 标注对应的 PRD 页面编号和功能点 |

---

## §3 各子节格式要求

### 3.1 信息架构

**内容**：页面包含的所有元素及其层级关系。
**格式**：树状结构或列表。
**边界**：不写具体尺寸、不写颜色、不写具体文案。

### 3.2 状态机

**内容**：页面在不同状态下的展示和行为。
**必须包含**：默认状态、空状态、加载状态、错误状态、成功状态。
**格式**：每个状态包含触发条件、视觉表现、可执行操作。
**边界**：不写 Loading 动画样式。加载/错误/成功状态如与顶层定义一致，只写引用。

### 3.3 页面流程

**内容**：用户从进入页面到离开的操作路径。
**格式**：流程图或步骤列表。
**边界**：不写页面内部交互细节、不写过渡动画。

### 3.4 异常处理

**内容**：页面可能遇到的异常情况及系统响应。
**格式**：表格，列：异常 | 触发条件 | 系统响应 | 用户感知。
**边界**：不写技术实现。至少覆盖 1 个异常，复杂页面建议 3 个。

### 3.5 与 PRD 对应

**内容**：标注本页面对应的 PRD 页面编号和功能点。
**格式**：PRD 页面编号、关联功能点、PRD 章节。
**边界**：每个交互页面必须对应 PRD §4 的一个页面编号。

---

## §4 格式规范

### 4.1 章节层级

- 一级：`## §N 页面名`
- 二级：`### N.M 子节名`
- 三级：`#### 细节描述（少用）`
- 禁止出现 `## §1.6` 这种同级但编号暗示子级的情况

### 4.2 页面编号

- 使用 PRD §4 定义的页面编号（如 `PAGE-001`, `PAGE-002`）
- 章节标题中标注：`## §1 {页面编号} {页面名}`

## §5 检查清单

### 一致性检验

- [ ] **结构一致性** `[脚本]`：每个页面包含 5 个子节（信息架构、状态机、页面流程、异常处理、与 PRD 对应）
- [ ] **范围一致性** `[模型]`：所有页面编号在上游 PRD 页面清单内；所有功能点在上游 PRD 功能清单内
- [ ] **格式一致性** `[脚本]`：页面编号使用 PRD §4 定义的编号；状态值与项目交互-顶层定义一致；手势规则符合项目交互-顶层定义
- [ ] **语义一致性** `[模型]`：状态流转与上游 PRD 业务规则一致；异常场景覆盖上游 PRD 定义的错误码触发条件；组件行为符合项目交互-顶层定义
- [ ] **内部自洽性** `[模型]`：本文档涉及的所有页面编号在「与 PRD 对应」子节中都有标注；状态机定义的状态值与异常处理中的触发条件不矛盾；同一流程中手势规则前后一致
- [ ] **跨文档链路一致性** `[模型]`（如已有下游文档）：每个页面在下游 UI 设计稿中有对应视觉定义；每个异常场景引用的错误码在下游技术方案接口设计中有映射

### 完整性检验

- [ ] 每个页面都有默认状态、空状态、加载状态、错误状态、成功状态的描述 `[脚本]`
- [ ] 每个操作流程有明确的前置条件、步骤、后置状态 `[模型]`
- [ ] 每个异常处理有触发条件、提示文案、恢复路径 `[模型]`
- [ ] 每个异常处理引用的错误码在 PRD §7 中真实存在 `[脚本]`
- [ ] 所有跳转目标页面要么在本文档范围内，要么正确引用外部文档 `[脚本]`
