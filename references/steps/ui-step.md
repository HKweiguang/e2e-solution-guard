# UI 设计模板

> 定位：定义产品长什么样，包括视觉风格、组件样式、页面布局、文案、色彩、字体、间距。

---

## §1 文件头部格式

```markdown
# UI设计-{页面名/设计稿名}

**覆盖功能点**：{功能编号}, {功能编号}, ...
**覆盖页面**：{页面编号}, {页面编号}, ...
**版本号**：v{主版本号}.{补丁号}
**制定日期**：YYYY-MM-DD
**作者**：姓名

**上游文档**：
> 以下仅为该步骤常见依赖的示例，实际生成时需根据项目上下文动态判断，不要机械套用。

| 文档 | 类型 | 引用范围 |
|------|------|---------|
| 交互设计-{页面组名} | 设计输入 | {功能编号}, {功能编号}, {页面编号范围} |
| 项目 UI-顶层定义 | 规范继承 | 色彩、字体、组件系统 |
```

---

## §2 章节结构

| 章节 | 说明 |
|------|------|
| §1 顶层定义引用 | 声明继承项目 UI-顶层定义 |
| §2 页面组件 | 页面特有的组件样式（可选） |
| §3 组件规范 | 通用组件的样式标准 |
| §4 页面设计 | 每个页面的视觉设计描述 |

---

## §3 各章节格式要求

### 3.1 顶层定义引用

**内容**：声明本页面 UI 稿继承项目 UI-顶层定义，不重复定义顶层视觉系统。
**格式**：自由文本，1-2 段。
**边界**：不写色彩、字体、间距的全局定义。

### 3.2 页面组件

**内容**：仅当某组件为**当前页面独有、不可复用**时在此定义。全局组件库中已定义的组件，以及可在模块内多页面复用的组件，均不在此处定义。
**格式**：每个组件包含用途、结构、样式（引用 Token）、状态。
**边界**：全局已有组件和模块级可复用组件不在此处重复定义。

### 3.3 组件规范

**内容**：本模块需要使用的全局组件清单，以及全局组件库中**未覆盖**的模块特有组件/变体定义。
**格式**：表格，列：组件名称 | 宽度 | 高度 | 圆角 | 背景色 | 文字色 | 说明。
**规范**：
- 全局组件库中已定义的组件：仅列出名称和引用 Token，不重复写样式数值
- 模块特有组件/变体：必须逐项定义样式，且 Token 必须在项目 UI-顶层定义中有定义，直接引用 Token 名
**边界**：不写组件的交互行为；不写全局已有组件的重复定义。

### 3.4 页面设计

**内容**：每个页面的视觉设计描述。
**格式**：从上到下的布局描述，使用 Token 和具体数值。
**边界**：每个页面必须有默认状态、空状态、错误状态的 UI 描述。不写交互行为、不写业务规则。

---

## §4 格式规范

### 4.1 尺寸单位

- 使用 px（Web）或 pt（iOS）/ dp（Android）
- 同一文档内单位统一

### 4.2 颜色格式

- 使用 HEX 格式
- 透明度使用 rgba

### 4.3 文案规范

- 必须与交互设计逐字一致
- 禁止自行修改文案
- 发现不一致 → 停止生成，询问用户以哪个为准

## §5 检查清单

### 一致性检验

- [ ] **结构一致性** `[脚本]`：§1–§4 章节齐全；§4 每个页面包含默认/空/错误三种状态描述
- [ ] **范围一致性** `[模型]`：所有页面在上游交互设计覆盖的页面范围内
- [ ] **格式一致性** `[脚本]`：颜色使用项目 UI-顶层定义 Token；尺寸使用项目 UI-顶层定义间距 Token；文案与交互设计逐字一致
- [ ] **语义一致性** `[模型]`：组件状态（默认/按下/禁用/错误）与交互设计定义的状态流转匹配；颜色使用符合项目 UI-顶层定义语义定义
- [ ] **内部自洽性** `[脚本]`：§4 中使用的每个 Token 在 §1 顶层定义引用或 §3 组件规范中都有出处；同一文档内颜色/尺寸单位统一；文案与交互设计逐字一致（发现不一致立即停止并询问）
- [ ] **跨文档链路一致性** `[模型]`（如已有下游文档）：§4 中每个页面的状态描述与下游技术方案接口的响应状态定义一致

### 完整性检验

- [ ] §4 每个页面有默认状态、空状态、错误状态的 UI 描述 `[脚本]`
- [ ] §3 每个组件引用了项目 UI-顶层定义中的 Token `[脚本]`
- [ ] §2 页面组件 + §3 组件规范覆盖所有交互设计中涉及的组件 `[模型]`
- [ ] §4 中每个组件的使用在交互设计中有对应定义 `[脚本]`
