## §1 P001 商品列表

### 1.1 信息架构

- 页面
  - 顶部搜索栏（固定）
    - 搜索输入框
    - 搜索按钮
  - 分类筛选栏（横向滚动）
    - 全部 / 文学 / 科技 / 生活 / 少儿（可左右滑动）
  - 商品列表（垂直滚动）
    - 商品卡片（重复）
      - 封面图
      - 书名
      - 作者
      - 价格
      - 销量标签
  - 底部 Tab 栏（全局）

### 1.2 状态机

| 状态 | 触发条件 | 视觉表现 | 可执行操作 |
|------|---------|---------|-----------|
| 默认 | 页面正常加载 | 展示商品卡片列表 | 搜索、筛选、点击商品 |
| 空状态 | 当前分类下无商品 | 居中插画 + 文案「暂无商品」+ 「去首页看看」按钮 | 点击按钮跳转首页 |
| 加载状态 | 下拉刷新或上拉加载更多 | 顶部/底部出现加载指示器（参见项目交互-顶层定义 2.4.2） | 等待加载完成 |
| 错误状态 | 网络请求失败 | 居中错误图标 + 文案「加载失败，点击重试」 | 点击重试 |
| 成功状态 | 数据加载完成 | 商品列表完整展示 | 正常交互 |

### 1.3 页面流程

```
进入页面 → 展示加载状态 → 加载成功展示默认状态
   ↓                          ↓
点击搜索栏              点击商品卡片
   ↓                          ↓
跳转搜索页              跳转 P002 商品详情
```

### 1.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 网络超时 | 首次加载超过 10 秒 | 自动重试 1 次，仍失败进入错误状态 | 看到错误页面，可手动重试 |
| 下拉刷新冲突 | 用户在下拉刷新过程中上拉加载 | 忽略上拉操作，优先完成刷新 | 列表保持当前状态，无异常感知 |

### 1.5 与 PRD 对应

- PRD 页面编号：P001
- 关联功能点：F001（浏览商品）
- PRD 章节：§4 页面清单

## §2 P002 商品详情

### 2.1 信息架构

- 页面
  - 导航栏（固定）
    - 返回按钮
    - 分享按钮
  - 内容区（垂直滚动）
    - 商品大图轮播
    - 价格区
      - 现价
      - 原价（划线）
      - 促销标签
    - 基本信息区
      - 书名
      - 作者
      - 出版社
      - 评分
    - 操作区（固定底部）
      - 「加入购物车」按钮
      - 「立即购买」按钮

### 2.2 状态机

| 状态 | 触发条件 | 视觉表现 | 可执行操作 |
|------|---------|---------|-----------|
| 默认 | 页面正常加载 | 完整展示商品信息 | 轮播图切换、加入购物车、立即购买 |
| 空状态 | 商品已下架或删除 | 居中插画 + 「商品已下架」 | 返回上一页 |
| 加载状态 | 页面初始化 | 骨架屏（参见项目交互-顶层定义 2.4.2） | 等待加载 |
| 错误状态 | 网络请求失败 | 错误提示 + 重试按钮 | 点击重试 |
| 成功状态 | 数据加载完成 | 完整展示 | 正常交互 |

### 2.3 页面流程

```
进入页面 → 加载成功展示默认状态
   ↓
点击「加入购物车」
   ↓
按钮短暂变为「已添加」→ 购物车 Tab 角标 +1
   ↓
点击「立即购买」
   ↓
校验登录态 → 未登录跳转登录页 → 登录后跳转 P004 订单确认
                        → 已登录直接跳转 P004 订单确认
```

### 2.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 库存为 0 | 商品库存不足 | 「立即购买」按钮置灰，文案变为「暂时缺货」 | 无法点击购买，提示缺货 |
| 未登录购买 | 未登录用户点击「立即购买」 | 保存当前页面状态，跳转登录页 | 登录后自动回到本页并继续流程 |

### 2.5 与 PRD 对应

- PRD 页面编号：P002
- 关联功能点：F001（浏览商品）、F002（管理购物车）
- PRD 章节：§4 页面清单

## §3 P003 购物车

### 3.1 信息架构

- 页面
  - 导航栏
    - 标题「购物车」
    - 管理按钮（切换编辑模式）
  - 内容区
    - 商品列表（重复）
      - 选择框
      - 封面缩略图
      - 书名
      - 单价
      - 数量步进器（- / 数量 / +）
    - 空状态（无商品时展示）
  - 底部结算栏（固定）
    - 全选复选框
    - 合计金额
    - 「去结算」按钮

### 3.2 状态机

| 状态 | 触发条件 | 视觉表现 | 可执行操作 |
|------|---------|---------|-----------|
| 默认 | 购物车有商品 | 展示商品列表，结算栏显示合计 | 选择/取消选择、修改数量、删除、结算 |
| 空状态 | 购物车无商品 | 居中插画 + 「购物车是空的」+ 「去逛逛」按钮 | 点击按钮跳转 P001 |
| 编辑状态 | 点击「管理」 | 商品项显示删除按钮，结算栏变为「删除」按钮 | 选择商品后点击删除 |
| 加载状态 | 页面初始化 | 骨架屏 | 等待加载 |
| 错误状态 | 请求失败 | 错误提示 + 重试 | 点击重试 |

### 3.3 页面流程

```
进入页面 → 有商品展示默认状态，无商品展示空状态
   ↓
点击「去结算」
   ↓
校验是否有选中商品
   ↓
有选中 → 校验登录态 → 跳转 P004 订单确认
无选中 → Toast 提示「请选择商品」
```

### 3.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 商品下架 | 购物车中的商品已下架 | 该商品项置灰，显示「已下架」，不可选中 | 视觉上区分，无法结算该商品 |
| 库存变化 | 商品库存小于购物车数量 | 数量自动调整为当前库存最大值，Toast 提示库存不足 | 数量被自动修正 |

### 3.5 与 PRD 对应

- PRD 页面编号：P003
- 关联功能点：F002（管理购物车）
- PRD 章节：§4 页面清单

## §4 P004 订单确认

### 4.1 信息架构

- 页面
  - 导航栏
    - 返回按钮
    - 标题「确认订单」
  - 内容区
    - 收货地址卡片
      - 收件人姓名/电话
      - 完整地址
      - 右箭头（可修改）
    - 商品清单区
      - 店铺名
      - 商品项（缩略图、书名、单价、数量）
      - 小计金额
    - 优惠区（可选）
    - 支付方式选择
      - 微信支付 / 支付宝 / 银行卡（单选）
  - 底部提交栏（固定）
    - 实付金额
    - 「提交订单」按钮

### 4.2 状态机

| 状态 | 触发条件 | 视觉表现 | 可执行操作 |
|------|---------|---------|-----------|
| 默认 | 页面正常加载 | 展示地址、商品、支付方式 | 修改地址、选择支付方式、提交订单 |
| 地址缺失 | 用户无默认收货地址 | 地址卡片显示「请添加收货地址」+ 添加按钮 | 点击添加地址 |
| 加载状态 | 页面初始化 | 骨架屏 | 等待加载 |
| 错误状态 | 请求失败 | 错误提示 + 重试 | 点击重试 |
| 提交中 | 点击「提交订单」 | 按钮变为加载态，显示「提交中...」 | 等待提交结果 |
| 提交成功 | 订单创建成功 | 跳转支付页 | — |
| 提交失败 | 库存不足或其他校验失败 | 按钮恢复，Toast 提示具体原因 | 根据提示调整 |

### 4.3 页面流程

```
进入页面 → 加载默认状态
   ↓
点击「提交订单」
   ↓
校验地址、库存 → 通过 → 创建订单 → 跳转支付页
              → 不通过 → Toast 提示失败原因
```

### 4.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 库存不足 | 提交时库存不足 | 订单创建失败，Toast 提示「商品库存不足，请减少数量」 | 返回商品详情或购物车调整 |
| 地址无效 | 收货地址超出配送范围 | 提交失败，提示「该地址暂不支持配送」 | 修改或新增地址 |

### 4.5 与 PRD 对应

- PRD 页面编号：P004
- 关联功能点：F003（提交订单）
- PRD 章节：§4 页面清单

## §5 P005 订单列表

### 5.2 状态机

| 加载状态 | 页面初始化或切换筛选 | 骨架屏 | 等待加载 |
| 错误状态 | 请求失败 | 错误提示 + 重试 | 点击重试 |

### 5.3 页面流程

```
进入页面 → 默认展示「全部」订单
   ↓
点击「待支付」筛选
   ↓
列表刷新，仅展示 PENDING 状态订单
   ↓
点击订单卡片
   ↓
跳转 P006 订单详情
```

### 5.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 列表加载失败 | 网络请求失败 | 错误状态，提供重试 | 点击重试重新加载 |
| 状态筛选无数据 | 某状态下用户无订单 | 空状态 | 提示该状态下无订单 |

### 5.5 与 PRD 对应

- PRD 页面编号：P005

## §6 P006 订单详情

### 6.1 信息架构

- 页面
  - 导航栏
    - 返回按钮
    - 标题「订单详情」
  - 内容区（垂直滚动）
    - 订单状态区（醒目展示当前状态）
    - 物流信息区（SHIPPED 及以上状态展示）
      - 物流公司 / 单号
      - 最新物流节点
    - 商品清单区
      - 商品项（缩略图、书名、单价、数量）
    - 金额明细区
      - 商品总额 / 运费 / 实付金额
    - 订单信息区
      - 订单编号 / 创建时间 / 支付方式
    - 收货地址区
  - 底部操作栏（根据状态动态展示）
    - PENDING：「取消订单」「立即支付」
    - PAID：无操作（等待发货）
    - SHIPPED：「确认收货」「查看物流」
    - COMPLETED：「申请售后」「再次购买」
    - CANCELLED：「再次购买」

### 6.2 状态机

| 状态 | 触发条件 | 视觉表现 | 可执行操作 |
|------|---------|---------|-----------|
| 默认 | 页面正常加载 | 完整展示订单信息 | 根据订单状态显示不同操作按钮 |
| 加载状态 | 页面初始化 | 骨架屏 | 等待加载 |
| 错误状态 | 请求失败 | 错误提示 + 重试 | 点击重试 |
| 操作确认 | 用户点击「取消订单」或「确认收货」 | 弹出 Dialog 二次确认 | 确认或取消 |

### 6.3 页面流程

```
进入页面 → 加载默认状态
   ↓
点击「取消订单」
   ↓
弹出确认 Dialog → 确认 → 调用取消接口 → 成功刷新页面状态为 CANCELLED
              → 取消 → 关闭 Dialog，状态不变
```

### 6.4 异常处理

| 异常 | 触发条件 | 系统响应 | 用户感知 |
|------|---------|---------|---------|
| 订单不存在 | 订单ID无效 | 错误状态，提示「订单不存在」 | 返回订单列表 |
| 取消失败 | 订单已支付或已取消 | Toast 提示「当前状态不可取消」 | 页面状态不变 |
| 无权查看 | 当前用户不是订单所有者 | 跳转错误页，提示「无权查看」 | 返回首页 |

### 6.5 与 PRD 对应

- PRD 页面编号：P006
- 关联功能点：F005（查看订单）、F006（取消订单）
- PRD 章节：§4 页面清单
