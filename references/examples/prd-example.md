## §1 模块概述

在线书店订单模块是用户完成购书交易的核心链路。用户从商品详情页发起购买，经过购物车确认、订单提交、支付到订单管理，全流程在本模块内闭环。本模块的核心用户价值是让用户能够快速、安全地完成图书购买，并随时跟踪订单状态。

本模块与商品模块的关系：商品模块提供商品信息和库存查询服务，订单模块通过接口调用获取商品数据，但不管理商品本身。与支付模块的关系：订单模块生成待支付订单后调用支付模块完成扣款，支付结果通过回调驱动订单状态变更。与物流模块的关系：订单状态变更为已支付后，物流模块接管发货和配送跟踪。

## §3 里程碑规划

当前版本 v1.0 包含 F001-F006 全部功能，覆盖用户从浏览商品到确认收货的完整购书流程。v1.0 目标是在一个迭代周期内上线核心交易链路，验证用户购买转化率。

后续方向：v1.1 计划增加订单评价功能（F007）、退换货申请（F008）、优惠券抵扣（F009）。v1.2 计划增加批量下单和预售订单支持。

## §5 数据模型

### 5.1 订单（order）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| order_id | 字符串 | PK, NOT NULL | 订单唯一编号，业务单号格式：ORD+时间戳+随机数 |
| user_id | 字符串 | FK, NOT NULL | 下单用户ID，关联用户表 |
| total_amount | Decimal | NOT NULL, ≥0 | 订单总金额，单位元，精确到分 |
| status | 字符串 | NOT NULL | 订单状态，枚举：PENDING / PAID / SHIPPED / COMPLETED / CANCELLED |
| pay_method | 字符串 | — | 支付方式：WECHAT / ALIPAY / CARD |
| receiver_address | 字符串 | NOT NULL | 收货地址完整文本 |
| created_at | 日期时间 | NOT NULL | 订单创建时间 |
| updated_at | 日期时间 | NOT NULL | 最后更新时间 |

### 5.2 订单项（order_item）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| item_id | 字符串 | PK, NOT NULL | 订单项唯一编号 |
| order_id | 字符串 | FK, NOT NULL | 所属订单，关联 order 表 |
| book_id | 字符串 | FK, NOT NULL | 书籍ID，关联商品表 |
| book_title | 字符串 | NOT NULL | 书籍标题快照，下单时固化 |
| quantity | 整数 | NOT NULL, >0 | 购买数量 |
| unit_price | Decimal | NOT NULL, ≥0 | 下单时单价快照，单位元 |
| subtotal | Decimal | NOT NULL, ≥0 | 小计金额 = quantity × unit_price |

## §6 业务规则

### 6.1 状态流转

订单生命周期状态机：

```
PENDING（待支付）
    │ 支付成功回调
    ▼
PAID（已支付）
    │ 商家发货
    ▼
SHIPPED（已发货）
    │ 用户确认收货 / 发货后7天自动确认
    ▼
COMPLETED（已完成）

PENDING（待支付）
    │ 用户取消 / 超时30分钟未支付
    ▼
CANCELLED（已取消）
```

**状态说明**：
- PENDING：订单已创建，等待支付，用户可取消
- PAID：支付成功，等待商家发货
- SHIPPED：商家已发货，用户可确认收货
- COMPLETED：订单完结
- CANCELLED：订单取消，不可恢复

### 6.2 权限规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 |
|---------|---------|---------|---------|
| R001 | 下单需登录 | 用户点击「立即购买」或「去结算」 | 未登录用户跳转登录页，登录成功后回到原页面继续流程 |
| R002 | 取消仅限本人 | 用户点击「取消订单」 | 校验当前用户ID与订单 user_id 一致，不一致返回无权操作错误 |
| R003 | 查看仅限本人 | 用户访问订单详情 | 校验当前用户ID与订单 user_id 一致 |

### 6.3 校验规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 |
|---------|---------|---------|---------|
| R004 | 库存校验 | 提交订单时 | 遍历订单项，逐条查询商品库存，任一商品库存不足则整单拒绝 |
| R005 | 金额校验 | 支付回调时 | 校验回调金额与订单 total_amount 一致，不一致视为支付异常，触发退款 |
| R006 | 时效校验 | 取消订单时 | 仅 PENDING 状态订单允许取消，其他状态拒绝 |

## §8 价值触达

**埋点事件**：
- `product_view`：商品详情页浏览
- `add_to_cart`：加入购物车
- `cart_checkout`：从购物车进入结算
- `order_submit`：提交订单成功
- `order_pay`：支付成功
- `order_cancel`：取消订单

**功能入口**：
首页 → 商品列表（P001）→ 商品详情（P002）→ 加入购物车 → 购物车（P003）→ 去结算 → 订单确认（P004）→ 提交订单 → 支付 → 订单列表（P005）

**通知触达**：
- 订单提交成功：站内信
- 支付成功：短信 + 站内信
- 商家发货：短信 + 站内信 + 推送
- 订单完成：站内信

**用户引导**：
- 新用户首次进入商品详情页，展示新人优惠券领取浮层
- 购物车为空时，展示「去逛逛」引导按钮

## §9 验收标准

| 验收项编号 | 功能点 | 验收项 | 条件 | 操作 | 预期结果 |
|-----------|--------|--------|------|------|---------|
| §9.001 | F001 | 商品列表筛选 | 用户进入商品列表页 | 点击「文学」分类 | 列表仅展示文学类商品 |
| §9.002 | F002 | 加入购物车 | 用户在商品详情页，库存充足 | 点击「加入购物车」，数量选择 2 | 购物车图标数量 +2，顶部 Toast 提示「已加入购物车」 |
| §9.003 | F003 | 提交订单成功 | 购物车有 1 件商品，用户已登录 | 点击「去结算」→ 确认地址 → 点击「提交订单」 | 生成订单，订单状态为 PENDING，跳转支付页 |
| §9.004 | F003 | 提交订单库存不足 | 购物车商品库存为 0 | 点击「去结算」→ 点击「提交订单」 | 提示「商品库存不足」，订单未生成 |
| §9.005 | F004 | 支付成功 | 订单状态为 PENDING | 完成支付流程 | 订单状态变为 PAID，收到支付成功通知 |
| §9.006 | F005 | 查看订单物流 | 订单状态为 SHIPPED | 进入订单详情页 | 展示物流公司和物流单号，显示最新物流节点 |
| §9.007 | F006 | 取消待支付订单 | 订单状态为 PENDING，用户为订单所有者 | 点击「取消订单」→ 确认 | 订单状态变为 CANCELLED，按钮变为「已取消」 |
| §9.008 | F006 | 取消已支付订单失败 | 订单状态为 PAID | 点击「取消订单」 | 提示「已支付订单不可取消」，状态不变 |
