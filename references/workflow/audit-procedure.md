# 审计执行策略

产物生成或修改完成后，运行审计确认产物质量。

---

## 硬性规则（AI 必须遵守，不可偏离）

1. **「审计」包含修复。** 指执行本文定义的完整闭环：机械检查→发现问题→修复→复验→语义检查→发现问题→修复→复验。禁止只检查不修复。
2. **一次只审计一份产物。** 禁止同时审计多份产物。
3. **机械未通过禁止启动语义检查。**
4. **修复后必须复验。** 机械修复→重新阶段一；语义修复→重新阶段二。
5. **删除优于标注。** 审计发现多余、错误、越界内容时，优先直接删除。例外：上游已修改但用户拒绝同步→标注 `[待同步]`。

---

## 核心原则

### 单向责任

每个产物在审计时**只对自己的上游负责**，不对下游负责。

- PRD 审计 → 检查 PRD 是否正确继承了顶层定义
- 交互设计审计 → 检查交互设计是否正确反映了 PRD
- UI 设计审计 → 检查 UI 是否正确实现了交互设计
- 技术方案审计 → 检查技术方案是否正确反映了 PRD/交互设计
- 测试审计 → 检查测试是否正确覆盖了 PRD/技术方案

**禁止**在单产物审计中验证下游实现。端到端一致性由独立的 cross-reference-check 负责。

### 覆盖声明完整性

产物头部声明的上游覆盖范围，必须与实际内容中引用的上游范围一致：
- **声明了就要覆盖**：头部列出的每个上游编号/章节，在实际内容中必须有对应引用
- **覆盖了就要声明**：实际内容中引用的每个上游编号/章节，必须在头部声明中列出



---

## 任务类型

| 任务 | 加载内容 | 适用场景 |
|------|---------|---------|
| **单产物审计** | 当前产物 + 直接上游文档 + 步骤模板 | 新建/修改一份产物后的常规审计 |
| **批量审计** | 目录扫描 + `audit-batch.md` | 审计目录或多个文件时 |
| **跨产物链路检查** | 两份上下游产物的关键章节 | 验证编号引用、接口映射、错误码对应 |

---

## 加载优先级

1. **当前产物**（全文）
2. **直接上游文档**（仅被当前产物引用的章节/段落，禁止全文加载）
3. **步骤模板**（当前产物类型的结构和内容要求）
4. **顶层定义**（仅与当前产物类型相关的章节）

**禁止加载**：下游产物、间接上下游全文。

---

## 分工边界

| 归属 | 职责 |
|------|------|
| **脚本** (`doc-audit.py`) | 格式、存在性、连续性、编号映射、计数验证——结果只有通过/不通过 |
| **AI** | 语义判断（正确性、合理性、满足度）——结果可能有"存疑" |

**规则**：能确定性描述的检查 → 脚本；需要业务上下文理解的检查 → AI。禁止降级规则精度让脚本覆盖。

---

## 执行流程

```
机械检查 → 机械未通过 → 修复 → 重新机械检查
   ↓ 通过
语义检查 → 语义未通过 → 修复 → 重新语义检查
   ↓ 通过
审计完成
```

### 阶段一：机械检查

```bash
SCRIPT="$HOME/.agents/skills/e2e-solution-guard/scripts/doc-audit.py"
python3 "$SCRIPT" <doc_path> --type <prd|interaction|ui|tech|test> \
  --upstream <upstream1.md> --top-level <top-level1.md>
```

- 无 blocking、无 warning → 通过，进入阶段二
- 存在 blocking 或 warning → 停止，修复后重新执行

### 阶段二：语义检查

**加载内容**：当前产物 + 直接上游文档 + 步骤模板（结构参照）+ **对应产物类型的语义检查清单**（`references/checklists/{type}-checklist.md`）

1. AI 读取检查清单，逐项判断，输出固定格式表格
2. 检查清单中的检查项必须全部覆盖，禁止跳过或压缩

**输出格式**：

| 检查点ID | 维度 | 层级 | 检查项 | 结果 | 证据 | 建议 |
|---------|------|------|--------|------|------|------|
| M-数据-L1-001 | 数据 | L1 | ... | 通过/不通过/存疑 | 产物原文 | 修正方向 |

**末行必须输出**：`语义审计完成：{N}/{total} 项已检查，0 项跳过。`

---

## 关键约束

- **[模型] 检查项的"列出"归脚本，"判断"归 AI**。AI 无权决定查几项，只决定每项的结果
- **语义审计输出必须是固定行数的表格**，少一行即视为不完整

---

## [模型] 项执行要求

1. **必须引用产物原文**：每项结论必须有具体代码/文本片段支撑，禁止笼统写"✅"
2. **必须量化**：颜色、尺寸、透明度必须给出具体数值
3. **禁止压缩**：不允许合并多项检查为一个结论，必须逐项独立输出
