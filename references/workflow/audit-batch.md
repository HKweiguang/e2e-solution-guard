# 批量审计指南

> 多文件批量审计的统一执行规则。单文件审计参见 [`audit-procedure.md`](audit-procedure.md)。

---

## 硬性规则（AI 必须遵守，不可偏离）

1. **一个 `/goal` 命令只对应一个文件。** 禁止打包目录、禁止列出多个文件、禁止"目录下所有 HTML 文件"这类表述。
2. **命令格式固定为 `/goal 审计 <完整文件路径>`。** 禁止展开流程细节（检查清单、机械检查命令、上下游验证维度等已在 skill 中定义，命令中禁止重复）。
3. **「审计」包含修复。** 指执行 `audit-procedure.md` 定义的完整闭环：机械检查→发现问题→修复→复验→语义检查→发现问题→修复→复验。禁止只检查不修复。
4. **递归扫描子目录。** 用户指定目录含子目录时，必须逐层递归列出每个文件的完整路径，每个文件单独一条命令。
5. **禁止在当前对话中直接执行批量审计。** 只输出命令序列，告知用户复制到终端执行，然后立即停止。

---

## 1. 规则总览

| 场景 | 执行方式 |
|------|---------|
| **单文件审计** | 直接启动审计流程，参见 [`audit-procedure.md`](audit-procedure.md) |
| **多文件批量审计** | **输出 `/goal` + `/goal next` 命令序列**，由用户在终端执行 |

---

## 2. 为什么批量审计不能直接执行

Kimi Code 平台可能将当前对话中的批量审计任务自动并行化到不同 agent。`/goal next` 队列通过"当前目标完成前 Agent 看不到后续目标"的机制，从物理层面杜绝并行。

---

## 3. 普通对话中的行为约束

当用户在**普通对话（非 `/goal` 模式）**中要求审计目录或多个文件时：

- 递归扫描目录（含子目录），列出每个待审计文件的完整路径
- 输出 `/goal` + `/goal next` 命令序列，每个命令对应一个文件
- 告知用户复制到终端执行
- **立即停止。禁止在当前对话中继续任何操作。**

---

## 4. 执行流程

```
阶段 1：当前对话（非 goal 模式）
  1. 递归扫描目录（含所有子目录）→ 列出每个待审计文件的完整路径清单
  2. 输出 `/goal` + `/goal next` 命令序列，每个命令对应一个文件
  3. 告知用户复制到终端执行
  4. **立即停止。禁止在当前对话中继续任何操作。**

阶段 2：终端执行队列
  /goal 审计 <文件1完整路径>
  /goal next 审计 <文件2完整路径>
  /goal next 审计 <文件3完整路径>
  ...
  └── Kimi 按队列顺序逐个执行，每完成一个自动出队下一个
```

### 4.1 错误示例 vs 正确示例

假设目录结构：
```
docs/03-UI设计/用户端-小程序/
├── M1-用户/
│   ├── 登录.html
│   └── 个人中心.html
└── M2-知识库/
    └── 首页.html
```

**❌ 错误**（打包多个文件到一个命令）：
```bash
/goal 审计 docs/03-UI设计/用户端-小程序/M1-用户/ 目录下所有 HTML 文件（登录.html、个人中心.html）...
/goal next 审计 docs/03-UI设计/用户端-小程序/M2-知识库/ 目录下所有 HTML 文件（首页.html）...
```

**✅ 正确**（每个文件单独一个命令，命令保持极简）：
```bash
/goal 审计 docs/03-UI设计/用户端-小程序/M1-用户/登录.html
/goal next 审计 docs/03-UI设计/用户端-小程序/M1-用户/个人中心.html
/goal next 审计 docs/03-UI设计/用户端-小程序/M2-知识库/首页.html
```

## 5. 生成清单

第一步都是生成清单：

```
/goal <目录路径> 列出该目录及子目录下所有需要审计的 Markdown/HTML 产物文件，
  按产物类型（PRD/交互设计/UI设计/技术方案/测试）分组，
  输出带编号的清单表格：编号 | 文件路径 | 产物类型 | 上游文档路径。

  只输出清单，不执行审计。
```
