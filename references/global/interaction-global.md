# 交互-全局规范

> 适用范围：定义跨模块共享的导航结构、交互规则、通用状态机和全局组件行为。模块级交互设计继承此全局规范。

**上游文档**：

| 文档 | 类型 | 引用范围 |
|------|------|---------|
| 项目 PRD 全局规范 | 规范继承 | §2.2 模块划分、§2.4 全局术语 |

---

## §1 章节结构

| 章节 | 必填 | 说明 |
|------|------|------|
| §1 导航结构 | 是 | Tab 栏、页面层级、路由规则 |
| §2 页面结构规范 | 是 | 通用布局、区域划分 |
| §3 全局手势规则 | 是 | 跨页面通用手势 |
| §4 通用状态机 | 是 | 登录态、加载态、网络异常等 |
| §5 全局组件行为 | 是 | Toast、Dialog、ActionSheet 等 |
| §6 跨页面数据同步 | 是 | 返回、刷新、状态保持规则 |

## §2 各章节写作要求

### 2.1 导航结构

#### 2.1.1 底部 Tab 栏
表格：序号、名称、对应模块。Tab 总数固定后不再增减。

#### 2.1.2 页面层级
一级/二级/三级，禁止四级及以上。标注允许/禁止三级页面的场景。

#### 2.1.3 路由规则
跳转方式（导航/重定向/替换）、传参规范、返回数据规则。

### 2.2 页面结构规范

#### 2.2.1 通用页面布局
标准区域划分：状态栏、导航栏、内容区、底部操作栏、Tab 栏。

#### 2.2.2 导航栏规范
返回按钮、标题、操作按钮的位置和规则。

#### 2.2.3 安全区适配
底部安全区和顶部状态栏的处理规则。

### 2.3 全局手势规则

表格：手势 | 适用场景 | 触发行为 | 禁止场景。每个手势必须明确适用场景和禁止场景。

### 2.4 通用状态机

#### 2.4.1 登录态状态机
明确必须登录和可匿名使用的功能，登录过期处理规则。

#### 2.4.2 页面加载状态机
加载中/成功/失败/空数据四种状态。明确各状态的展示方式和系统响应。

#### 2.4.3 网络异常状态机
自动重试或手动重试规则，连续请求失败的处理方式。

### 2.5 全局组件行为

每个组件包含：触发条件、展示规则、消失规则、禁止场景。组件包括：Toast、Dialog、ActionSheet、加载指示器。

### 2.6 跨页面数据同步

表格：场景 | 行为 | 数据刷新规则。明确返回刷新 vs 保持状态的区分规则。

## §3 内容边界

| 写在此文件中 | 写在模块交互设计中 |
|-------------|------------------|
| Tab 栏结构 | 模块内页面跳转流程 |
| 全局手势 | 模块特有手势 |
| 通用状态机 | 模块业务状态机 |
| 全局组件行为 | 模块页面布局细节 |
| 返回/刷新规则 | 模块内页面数据流 |

**红线**：禁止出现具体页面的字段定义、具体表单校验规则、具体业务状态流转（通用状态机除外）。

---

## §4 与模块交互设计的关系

- 模块交互设计开头声明：`继承项目交互全局规范的导航结构、手势规则和组件行为`
- 引用格式：`参见本规范 §{章节}「{规则}」`
- 模块交互设计中的页面必须遵循 §1 的导航层级
- 发现全局规则不适用 → 反馈给本规范维护者，禁止模块擅自覆盖
