## §2 各章节写作要求

### 2.1 色彩系统

#### 2.1.1 主色
默认/深色/浅色 3 层级。色值用 HEX 格式。

#### 2.1.2 语义色
成功/警告/错误/信息。每个语义色必须有明确的 UI 用途，禁止混用。

#### 2.1.3 中性色
文字主/次/禁用、分割线、背景、卡片背景。

#### 2.1.4 色彩使用规则
主按钮、次级按钮、输入框、禁用状态等场景的色彩映射。

### 2.2 字体系统

#### 2.2.1 字号层级
至少 4 级，不超过 6 级。相邻层级字号差距至少 2px。字重用数值。

#### 2.2.2 字体使用规则
页面标题、模块标题、正文、辅助说明等场景的字号/字重/颜色映射。

### 2.3 间距系统

#### 2.3.1 基础单位
所有间距必须为基础单位的整数倍。

#### 2.3.2 间距 Token
xs/sm/md/lg/xl/xxl 六级。

#### 2.3.3 页面边距
页面水平边距、列表项内边距、卡片内边距。

#### 2.3.4 圆角 Token
xs/sm/md/lg/full 五级。

### 2.4 阴影与边框

#### 2.4.1 阴影层级
sm/md/lg 三级。标注各层级的使用场景。

#### 2.4.2 边框规则
输入框、卡片、按钮等场景的宽度/颜色/圆角。

### 2.5 组件库

每个组件包含：尺寸、颜色、状态、间距。全局通用组件包括：
- **Button**：Primary / Secondary / Text 三种类型；状态：默认/按下/禁用/加载中
- **Input**：状态：默认/聚焦/错误/禁用
- **Card**、**List Item**、**Avatar**、**Badge**：状态：默认/按下/禁用/加载中

### 2.6 图标规范

图标尺寸（Tab 栏/导航栏/列表项/按钮内）、风格（线性/填充/双色）、颜色规则。

### 2.7 暗黑模式

Token | 亮色模式 | 暗色模式。仅列出与亮色模式不同的 Token。

## §3 内容边界

| 写在此文件中 | 写在模块 UI 稿中 |
|-------------|----------------|
| 色彩 Token 定义 | 具体页面配色组合 |
| 字体层级定义 | 具体页面文字排版 |
| 间距 Token 定义 | 具体页面布局间距 |
| 组件样式定义 | 组件在页面中的组合方式 |
| 圆角/阴影 Token | 具体元素的视觉效果 |

**红线**：禁止出现具体页面的布局结构、具体页面的文案内容、具体模块的自定义组件。

## §4 与模块 UI 稿的关系

- 模块 UI 稿开头声明：`继承项目 UI 全局规范的色彩、字体和组件系统`
- 使用全局 Token 时直接引用 Token 名，不重复写色值或像素值
- 组件必须从 2.5 组件库中选取，模块特有组件需在模块 UI 稿中注明
- 布局尺寸必须使用 2.3 的间距 Token
- 发现全局 Token 不够用 → 反馈给本规范维护者，禁止模块擅自新增
