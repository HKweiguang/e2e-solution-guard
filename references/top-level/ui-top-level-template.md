## §2 各章节写作要求

### 2.1 色彩系统

#### 2.1.1 主色
项目定义主色层级数及命名（常见如 2-5 级：默认/深色/浅色，或 50-900 等权重体系）。色值格式由项目定义（HEX / HSL / OKLCH / CSS 变量）。

#### 2.1.2 语义色
项目定义语义色集合（常见如成功/警告/错误/信息，项目可扩充或调整命名）。每个语义色必须有明确的 UI 用途，禁止混用。

#### 2.1.3 中性色
文字主/次/禁用、分割线、背景、卡片背景。

#### 2.1.4 色彩使用规则
主按钮、次级按钮、输入框、禁用状态等场景的色彩映射。

### 2.2 字体系统

#### 2.2.1 字号层级
项目定义字号层级数量及命名（常见 3-8 级，如 Display/H1-H6/Body/Caption/Overline 等）。相邻层级需有明显视觉区分。字重由项目定义（数值如 400/500/700，或名称如 Regular/Medium/Bold）。

#### 2.2.2 字体使用规则
页面标题、模块标题、正文、辅助说明等场景的字号/字重/颜色映射。

### 2.3 间距系统

#### 2.3.1 基础单位
所有间距必须为基础单位的整数倍。

#### 2.3.2 间距 Token
项目定义间距 Token 数量及命名（常见 3-8 级，命名如 xs/sm/md/lg/xl/xxl 或 4/8/16/24/32/48/64 等）。

#### 2.3.3 页面边距
页面水平边距、列表项内边距、卡片内边距。

#### 2.3.4 圆角 Token
项目定义圆角 Token 数量及命名（常见 3-6 级）。

### 2.4 阴影与边框

#### 2.4.1 阴影层级
项目定义阴影层级数量及命名（常见 3-5 级，如 Material Design 的 0/1/2/3/4/8dp）。标注各层级的使用场景。

#### 2.4.2 边框规则
输入框、卡片、按钮等场景的宽度/颜色/圆角。

### 2.5 组件库

每个组件包含：尺寸、颜色、状态、间距。以下组件清单为常见示例，项目根据实际技术栈和 UI 框架调整：
- **基础组件示例**：Button（按钮类型、各状态样式）、Input（各状态样式）、Card、List Item
- **状态说明**：状态定义由项目确定（如默认/悬停/按下/聚焦/禁用/加载中/错误等），不同平台状态可能不同（Web 有 hover/focus，移动端有按下态）

### 2.6 图标规范

图标尺寸（根据项目类型定义场景，如移动端为 Tab 栏/导航栏/列表项/按钮内，Web 为页头/侧边栏/行内/按钮内）、风格（线性/填充/双色/渐变等）、颜色规则。

### 2.7 暗黑模式

Token | 亮色模式 | 暗色模式。仅列出与亮色模式不同的 Token。

**红线**：禁止出现具体页面的布局结构、具体页面的文案内容、具体模块的自定义组件。

## §4 与模块 UI 稿的关系

- 模块 UI 稿开头声明：`继承项目 UI-顶层定义的色彩、字体和组件系统`
- 使用全局 Token 时直接引用 Token 名，不重复写色值或像素值
- 组件必须从项目 UI-顶层定义 2.5 组件库中选取，模块特有组件需在模块 UI 稿中注明
- 布局尺寸必须使用 2.3 的间距 Token
- 发现全局 Token 不够用 → 反馈给本规范维护者，禁止模块擅自新增
