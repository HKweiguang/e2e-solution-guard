# 交互设计检查清单

> 机械检查（格式、编号、SVG 元素数量）由 `doc-audit.py` 执行。以下为 AI 语义检查项。

---

## 输出格式要求（AI 必须遵守）

每项输出必须包含 5 个字段，缺一不可：

| 字段 | 要求 |
|------|------|
| **检查项** | 复制本清单中的检查项编号和名称 |
| **结论** | 通过 / 不通过 / 存疑（三选一） |
| **证据** | **必须引用产物原文片段**（页面编号、状态机描述、异常场景文本） |
| **判定依据** | 为什么是这个结论，引用本清单中的判定标准 |
| **建议** | 结论为"不通过"或"存疑"时必须给出具体修正方向 |

**禁止**：跳过检查项、合并多项为一个结论、无证据写"✅"。

## 对上游（PRD）

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| I-U-01 | 功能点覆盖 | **通过**：交互设计中每个页面/组件对应 PRD §3 至少一个功能点；PRD 中全部 P0 功能点在交互设计中都有对应。**不通过**：存在 P0 功能点在交互设计中无对应页面/组件 | 引用 PRD §3 中 P0 功能点列表 + 引用交互设计中对应页面/组件 |
| I-U-02 | 业务状态映射 | **通过**：状态机中的业务状态名称与 PRD §6 状态定义一致，状态转换条件与 PRD §6 状态流转条件一致（无矛盾、无遗漏）。**不通过**：状态名称不一致、转换条件矛盾、或 PRD 中定义的状态在状态机中缺失 | 引用 PRD §6 状态定义 + 引用交互设计状态机中对应状态和转换条件 |
| I-U-03 | 错误场景覆盖 | **通过**：交互设计异常处理覆盖了 PRD §7 中定义的全部错误码触发场景（每个错误码至少有一个异常处理）。**不通过**：存在 PRD §7 错误码在交互设计异常处理中无对应 | 引用 PRD §7 错误码列表 + 引用交互设计异常处理中对应场景 |
| I-U-04 | 用户故事对应 | **通过**：每个页面的交互流程能支撑其关联的 PRD 用户故事完成目标（从进入页面到完成目标的路径完整）。**不通过**：页面交互流程无法支撑用户故事完成（缺少关键操作步骤） | 引用 PRD §2 用户故事目标 + 引用交互设计页面流程，说明路径完整性 |
| I-U-05 | 权限规则体现 | **通过**：PRD §6 中定义的权限级别差异在交互设计中通过元素可见性/可操作性体现（如管理员可见但普通用户不可见的元素有明确标注）。**不通过**：权限差异在交互设计中完全被忽略，或未标注 | 引用 PRD §6 权限规则 + 引用交互设计中体现/未体现该权限的元素 |

## 单产物内业务语义

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| I-S-01 | 状态机完整性 | **通过**：每个页面状态机覆盖 6 种基础交互状态（默认/悬停/按下/聚焦/禁用/加载）+ 4 种异步状态（空/错误/成功/骨架）。**不通过**：缺失任何一种状态定义 | 引用状态机中已定义的状态列表，指出缺失项 |
| I-S-02 | 状态转换合理性 | **通过**：状态间转换条件逻辑自洽（加载态只能由默认态进入、错误态有明确恢复路径、不存在循环死锁）。**不通过**：存在状态转换条件矛盾、或无恢复路径的死状态 | 引用状态转换条件原文，逐条分析逻辑自洽性 |
| I-S-03 | 异常场景充分性 | **通过**：每个页面异常处理覆盖该页面可能发生的全部异常场景（网络失败/数据为空/权限不足/超时/第三方服务失败）。**不通过**：缺失典型异常场景（如列表页无空状态处理、表单提交无网络失败处理） | 引用异常处理列表，与页面可能发生的异常对比，指出遗漏 |
| I-S-04 | 页面流程闭环 | **通过**：页面流程中每个跳转都有明确的返回路径，用户从任何页面都能回到主干流程。**不通过**：存在"有去无回"的跳转（如弹窗关闭后无返回入口、深层页面无回到列表页的路径） | 引用页面流程描述，标出无返回路径的跳转 |
| I-S-05 | 多端差异声明 | **通过**：如存在多端差异（Web/App/小程序），差异点与业务场景相关（如支付流程在小程序中缺少某些步骤因平台限制）。**不通过**：差异点为无意义的平台差异（如"Web 用蓝色、App 用绿色"） | 引用多端差异声明原文，分析差异是否与业务场景相关 |
| I-S-06 | 力学约束合理性 | **通过**：超时/重试/防抖数值与业务场景匹配（支付操作重试间隔 ≥ 5s 防重复扣款，列表刷新防抖 300ms）。**不通过**：数值与业务场景明显不符（如支付操作重试间隔 100ms） | 引用力学约束数值 + 说明该数值与业务场景的匹配/不匹配原因 |

## 边界维度

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| I-B-01 | 无越界内容 | **通过**：交互设计中无颜色/字体/阴影等视觉样式描述、无技术实现细节。**不通过**：发现交互设计中包含视觉样式色值、CSS 类名、接口 URL 等不属于交互范畴的内容 | 引用越界内容原文 |
| I-B-02 | 页面结构图聚焦 | **通过**：SVG 页面结构图反映页面特有的功能结构（每个页面的元素标签不重复、能区分不同页面的功能差异）。**不通过**：所有页面复制同一套通用外壳模板（状态栏+导航栏+内容区域+Tab栏） | 引用两个不同页面的 SVG 结构图对比，指出重复/差异 |
