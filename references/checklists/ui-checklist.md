# UI 设计检查清单

> 机械检查（HTML 语法、CSS 变量存在性、DOCTYPE）由 `doc-audit.py` 执行。以下为 AI 语义检查项。

---

## 输出格式要求（AI 必须遵守）

每项输出必须包含 5 个字段，缺一不可：

| 字段 | 要求 |
|------|------|
| **检查项** | 复制本清单中的检查项编号和名称 |
| **结论** | 通过 / 不通过 / 存疑（三选一） |
| **证据** | **必须引用 HTML/CSS 代码片段**（具体 class 名、CSS 属性值、section id） |
| **判定依据** | 为什么是这个结论，引用本清单中的判定标准 |
| **建议** | 结论为"不通过"或"存疑"时必须给出具体修正方向 |

**禁止**：跳过检查项、合并多项为一个结论、无代码片段写"matches upstream"。

---

## 对上游（交互设计）

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| U-U-01 | 页面覆盖完整性 | **通过**：交互设计声明的每个页面在 HTML 中都有对应 `<section id>`，无遗漏页面。**不通过**：存在交互设计声明的页面在 HTML 中无对应 section | 引用交互设计页面列表 + 引用 HTML 中 section id 列表，逐页比对 |
| U-U-02 | 组件覆盖完整性 | **通过**：交互设计每个页面声明的组件/模块在 HTML 中都有对应结构，无遗漏组件。**不通过**：存在交互设计中声明的组件在 HTML 中缺失 | 引用交互设计中组件声明 + 引用 HTML 中对应组件结构 |
| U-U-03 | 状态映射正确性 | **通过**：交互设计定义的 10 种交互状态在 HTML 中都有对应 CSS 类/伪类（:hover/:active/:focus/.disabled/.loading/.empty/.error/.success/.skeleton）。**不通过**：缺失任何一种状态的 CSS 实现 | 引用交互设计状态矩阵 + 引用 HTML/CSS 中对应状态类名 |
| U-U-04 | 状态视觉语义 | **通过**：每种状态的视觉表现与其业务语义匹配（错误态使用 `--color-error`、禁用态有明显区分、加载态保留上下文结构）。**不通过**：错误态与默认态视觉几乎相同、禁用态无法区分、加载态空白无占位 | 引用各状态的 CSS 代码片段，分析视觉差异 |
| U-U-05 | 区域结构对应 | **通过**：HTML 区域结构（section/div 层级）与交互设计 SVG 页面结构图的区域划分一致。**不通过**：HTML 区域划分与交互设计明显不符 | 引用 SVG 页面结构图的区域描述 + 引用 HTML 中对应 section/div 层级 |
| U-U-06 | 响应式一致性 | **通过**：响应式结构变更与交互设计 §4.9 声明的断点和布局策略一致。**不通过**：断点设置与交互设计声明不符、或缺少交互设计声明的断点 | 引用交互设计 §4.9 断点声明 + 引用 HTML 中 @media 查询代码 |

## 对上游（PRD / UI-顶层定义）

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| U-U-07 | Token 一致性 | **通过**：CSS 变量命名与 UI-顶层定义 Token 命名一致，状态映射表中的 Token 在 UI-顶层定义中已声明。**不通过**：CSS 变量命名不一致、或使用了未声明的 Token | 引用 UI-顶层定义 Token 命名 + 引用 HTML/CSS 中使用的变量名 |
| U-U-08 | 错误码视觉映射 | **通过**：错误状态的视觉表现与 PRD §7 错误码严重程度匹配（致命错误 vs 警告性错误有视觉区分）。**不通过**：所有错误状态使用同一种视觉样式，无严重程度区分 | 引用 PRD §7 错误码严重程度分类 + 引用错误态 CSS 代码 |

## 单产物内业务语义

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| U-S-01 | 状态差异可感知 | **通过**：普通用户能区分默认态/禁用态/错误态/加载态（视觉差异足够明显，不依赖颜色单一属性）。**不通过**：状态间仅靠细微色差区分、或 disabled 状态与默认态几乎相同 | 引用各状态 CSS 代码，计算/描述具体视觉差异 |
| U-S-02 | 无障碍合规 | **通过**：颜色对比度满足 WCAG AA（正文 ≥ 4.5:1），状态信息不单独依赖颜色传达（必须配合图标或文字）。**不通过**：对比度不足、或仅用颜色区分错误/成功状态 | 引用具体色值，计算对比度 |

## 边界维度

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| U-B-01 | 无越界内容 | **通过**：HTML 中无 JavaScript 业务逻辑、无数据请求、无后端接口调用。**不通过**：发现 HTML 中包含 JS 业务逻辑、fetch/XHR 调用、或后端相关的描述 | 引用越界代码片段 |
