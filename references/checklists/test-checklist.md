# 测试检查清单

> 机械检查（格式、编号、表格列完整性）由 `doc-audit.py` 执行。以下为 AI 语义检查项。

---

## 输出格式要求（AI 必须遵守）

每项输出必须包含 5 个字段，缺一不可：

| 字段 | 要求 |
|------|------|
| **检查项** | 复制本清单中的检查项编号和名称 |
| **结论** | 通过 / 不通过 / 存疑（三选一） |
| **证据** | **必须引用测试用例原文片段**（用例编号、测试步骤、预期结果、关联的验收标准/错误码） |
| **判定依据** | 为什么是这个结论，引用本清单中的判定标准 |
| **建议** | 结论为"不通过"或"存疑"时必须给出具体修正方向 |

**禁止**：跳过检查项、合并多项为一个结论、无证据写"覆盖完整"。

## 单产物内业务语义

| 编号 | 检查项 | 判定标准 | 强制引用 |
|------|--------|---------|---------|
| S-S-01 | 测试场景充分性 | **通过**：异常测试覆盖了参数非法/权限不足/数据不存在/网络异常/并发冲突/第三方故障六类典型异常。**不通过**：缺失其中 1 类或多类异常场景 | 引用测试 §2 异常场景列表，与六类典型异常逐类比对 |
| S-S-02 | 测试数据合理性 | **通过**：测试数据使用描述性占位符（非硬编码真实数据），覆盖边界值（空值/最大值/特殊字符）。**不通过**：使用硬编码真实数据、或未覆盖边界值 | 引用测试 §1 中测试数据示例，分析是否覆盖边界值 |
| S-S-03 | 恢复验证 | **通过**：系统异常和第三方异常测试包含恢复验证（异常恢复后系统能正常继续，数据状态正确）。**不通过**：只测试了异常触发，未验证异常恢复后的状态 | 引用测试 §2 中包含恢复验证的用例，指出未包含恢复的用例 |
| S-S-04 | 覆盖检查真实性 | **通过**：§6 覆盖检查报告真实反映了测试覆盖情况，未覆盖项有合理的未覆盖原因（如"该功能在 v2 实现，当前版本不涉及"）。**不通过**：未覆盖项无原因、或原因不合理 | 引用 §6 覆盖检查报告中未覆盖项及其原因，分析合理性 |
