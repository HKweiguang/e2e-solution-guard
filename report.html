🏗️ 产物组织：先角色，后设备

角色和设备不是同一级。先按角色分文件，再在角色内按设备分。

角色 → 设备 📁 用户管理/ ├── prd.md ← 统一 PRD，所有角色功能点 ├── 交互设计-C端.md ← C 端交互 ├── 交互设计-管理后台.md ← 管理后台交互 ├── ui-C端-web.html ← C 端 Web ├── ui-C端-app.html ← C 端 App ├── ui-管理后台-web.html ← 管理后台 Web ├── 技术方案.md ← 统一技术方案 └── 测试报告.md
📌 一个模块一个 PRD 📌 交互设计按角色分 📌 UI 按角色+设备分 📌 技术方案统一

📐 顶层定义 = 通用规则仓库

跨模块共享的规则写在顶层定义里，模块级产物直接引用，禁止擅自扩展。

🎮 交互-顶层定义

1交互令牌 — 状态色、焦点环、动效分级
2状态规范 — 6 基础 + 4 异步状态
3反馈规范 — 时效红线、动效分级、缓动曲线
4交互规范 — 导航、页面层级、手势、组件行为
5平台交互规范 — Web/App/小程序交互差异

🎨 UI-顶层定义

1设计令牌 — Token 命名、三级映射
2色彩系统 — 调色板、色阶、色彩角色
3字体系统 — 家族、字号层级、字重
4间距与布局 — 基础单位、栅格、断点
5组件库 — 分类、全局状态、清单
6平台视觉规范 — Web/App/小程序视觉差异

📱 平台差异：角色内不同设备的交互/视觉差异

同一角色下，Web、App、小程序的交互和视觉差异，在顶层定义中统一声明。

🌐 Web 端

  • 鼠标 hover / 右键菜单
  • 键盘 Tab 聚焦
  • Ctrl+S / Ctrl+F 快捷键
  • 触摸支持（二合一）

📱 App 端

  • 点击 / 长按 / 滑动 / 双指缩放
  • 系统返回键 / 顶部返回
  • 软键盘 / 语音 / 扫码
  • 推送 / 本地通知

💬 小程序

  • 右上角分享 / 按钮分享
  • 下拉刷新
  • 调用扫码能力
  • 页面生命周期

📤 改上游

自动扫描所有下游产物，列出受影响清单。要求同步更新，或标注 [待同步]。

📥 改下游

逐条比对是否超出上游范围。超出 → 先改上游，禁止擅自扩展。

🤖 自动审计

检查项 查什么 例子
编号连续性 跳号 / 重复 USER-001 后直接 USER-003
上游缺失 有无声明引用 交互设计没写引用了哪个 PRD
跨产物对齐 下游是否漏了上游编号 PRD 有 USER-002，交互没提到
接口一致性 §4 和 §13 数量是否一致 §4 写了 5 个接口，§13 列了 4 个
HTML 完整性 UI 稿是否合格 没 DOCTYPE / 没 upstream / 写了 onclick
边界违规 越界写了不该写的 交互设计里写"按钮用 #ff0000"

🚦 两条红线

⛔ 改上游必须通知下游

禁止改了上游，下游还停留在旧版本

⛔ 改下游不能超出上游

想加新功能？先改 PRD

📋 一句话

类比：e2e-solution-guard 像建筑行业的图纸管理系统——确保每份图纸有编号、知道从哪份总图派生、改了总图自动通知相关图纸、施工队永远拿到最新版。价值不在"写得多快"，而在"改的时候不慌"。
